# Earnings Workflow

This project uses three related earnings workflows adapted from finance-skills concepts.

## Earnings preview

Before an earnings event:

- identify next earnings date if available
- summarize consensus EPS / revenue estimates if available
- show estimate trends or revision direction if available
- mention prior beat/miss context if available
- list missing data explicitly

## Earnings recap

After an earnings event:

- compare actual EPS vs estimate
- compare actual revenue vs estimate
- compute or display surprise magnitude when available
- describe near-term price reaction only if price history exists
- avoid unsupported storytelling

## Estimate analysis

For analyst estimate workflows:

- summarize current-quarter and next-quarter EPS / revenue expectations
- summarize current-year and next-year expectations when available
- show recommendation/target metadata as context, not instruction
- treat sparse coverage conservatively

## Guardrails

- do not infer actual earnings results when only estimates exist
- do not treat analyst estimates as a recommendation
- do not hide unavailable fields; expose them as missing
